Lab 7 - JavaScript, JSON and AJAX

In lab 3 we made a website which consisted of a homepage (index.html) and a projects page. Since then we have been adding projects to that page for each lab.

Well, here we go again. We are going to modify the projects page we created in that lab again, but this time in a different way.

In this lab,

  • We are going to make a JSON file with a list of our project menu items

    I have provided a JSON template to get you started.

    I have provided a sample of Lab 7 from a previous year as an example of how to use AJAX to read a file

  • We are going to write some JavaScript to read that file and create our projects menu for our website from that file.

    Your javascript file must be external to your HTML file.

In order to test this file with Chrome when the website is loaded on your PC/Mac, you will need to open Chrome with one of the commands shown below, and as shown on LMS a few weeks ago.

  • On the PC:
    • chrome.exe --allow-file-access-from-files
  • on the Mac:
    • open -b com.google.chrome --args --allow-file-access-from-files
    • -or-
    • /Applications/Google Chrome.app/Contents/MacOS/Google Chrome --allow-file-access-from-files

It may be possible to bypass this need by using a .js extension for your json file name, but this is not always consistent.

The reason for this has to do with the Same Origin Policy protection in Chrome

You will not need these flags when running your site from RCS.

When you are done, place a link to the Homepage of your website into a readme file in the root of your lab7 folder prior to zipping it up and submitting on LMS.
